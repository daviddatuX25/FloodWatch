Overview

What is FloodWatch?

FloodWatch is a real-time flash flood detection and early warning system deployed along canals and waterways. It uses a network of low-cost sensor nodes that communicate wirelessly via LoRa radio in a multi-hop relay chain — passing data to a base station that runs machine learning inference, syncs to the cloud, and sends SMS alerts.

The prototype demonstrates a minimum two-node deployment. All field nodes share identical hardware — role is defined by attached sensors and position in the network. Alerts reach two audiences: field maintenance crews on-site, and official responders (DRRMO/barangay) via SMS and dashboard.

📡

Multi-hop LoRa Network

Nodes relay data hop-by-hop — only one internet connection needed at the base station.

🧠

ML-Powered Surge Detection

Edge Impulse model classifies normal flow vs. anomalous surge — smarter than simple thresholds.

📱

SMS + Web Alerts

Automatic SMS to barangay officials and DRRMO, plus a live web dashboard with 3D canal simulation.

🚧

Canal Health Monitoring

Detects blockages by correlating rising water level with dropping flow rate — alerts maintenance crews before a clog escalates into a flood.

Alert System

Alert Levels

Flood Alerts

Yellow — Rise > 5cm in 10 minutes

Early warning. Water is rising faster than normal.

Short beep · Yellow LED SMS → tanod + DRRMO officer
Orange — Rise > 15cm in 10 minutes

Significant rise. Residents should prepare to evacuate.

Fast double beep · Orange LED SMS → barangay captain + resident list
Red — Rise > 30cm in 10 minutes AND water ≥ MDRRMO evacuation height

Both conditions must be met. The MDRRMO absolute height gate prevents false Red alerts — rapid rise that stays well below official evacuation level will not trigger Red.

Continuous alarm · Red LED SMS all contacts + dashboard alarm MDRRMO height gate required

Canal Clog Detection

Detected when water level rises while flow rate drops — indicating upstream blockage, not rainfall. A maintenance issue that can escalate into a flood if uncleared.

Clog — Water rising + flow rate dropping (Node A only)

Separate from flood alert levels. Does not imply evacuation. Correct response: send a maintenance crew to clear the blockage.

Slow triple beep · Blue LED SMS → maintenance immediately Escalates to DRRMO if unresolved

Risks

Risk Assessment

LoRa signal blockage

Site survey required; 433MHz penetrates obstacles better than higher bands.

Solar shading

Node B is battery-only; spare cells maintained by barangay for shaded sites.

Node submersion

Mount above flood height; IP65 enclosure used for all field nodes.

WiFi unavailability

A7680C 4G module planned as modular fallback for base station connectivity.

LoRaMesher instability

Pre-deployment routing tests; linear relay fallback in case mesh fails.

ML false positives

ML flags cross-checked with thresholds before any alert is issued.

False Red alerts

Red requires both rate-of-rise AND MDRRMO absolute height gate — dual condition eliminates most false alarms.

Insufficient ML training data

Threshold-only launch; ML activates after 2–4 weeks of labeled data.

Scope

Prototype Scope & Scalability

The prototype validates the complete pipeline across two nodes — including dual-pipeline alert architecture, actuator control via LoRa downlink, clog detection, and MDRRMO-gated Red alerts.

Each additional node costs approximately ₱886 bare minimum (includes buzzer + LED). Full waterway coverage requires a formal site survey to determine node spacing and count.

⚡

Power Efficiency

~15mA average draw at 30s intervals = 146 hours standalone. Buzzer and LEDs only draw during alert states — negligible on average power budget.

🔧

Hardware Modularity

Any node can be upgraded with additional sensors without changing the base board or reflashing firmware.

🔮

Future Enhancements

4G LTE fallback, PAGASA forecast API, mobile app, cloud multi-station view, public resident dashboard (pending MDRRMO approval).
